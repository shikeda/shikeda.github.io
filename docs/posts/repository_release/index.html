    Repository Release

    May 21, 2025

    krm関連のファイルを別のリポジトリに移す #

    ChatGPTに教えてもらった手順をメモ。ほぼそのままコピペです。

    前提の整理 #

    旧構成（これまで）：

    https://github.com/shikeda/HDIC
├── README.md
├── TSV.tsv
├── KTB.tsv
├── ...
├── TSJ_ndl.tsv
├── YQF.tsv
├── images
│   └── ...
└── v1.2
    ├── LICENSE
    ├── README.md
    ├── README_jp.md
    ├── krm_main.json
    ├── krm_main.tsv
    ├── krm_notes.json
    ├── krm_notes.tsv
    ├── ...
    ├── krm_wakun.json
    └── krm_wakun.tsv

    新構成（案）：

    別のリポジトリ shikeda/krm を作成し、そこでバージョン（v1.2, v1.3…）を Git のタグやリリース機能で管理。

    1. 「Git のタグ」や「リリース」とは？ #

    Gitでは、特定の時点（コミット）に名前を付けて「この時点が v1.2 です」と記録できます。これを**タグ（tag）**といいます。

    GitHub では、タグに説明文を加えたものを「リリース（Release）」として公開できます。

    これにより：

    • 過去バージョンに戻りたいとき、簡単に切り替えられる
    • v1.2 のファイルだけをダウンロードしたい人に、リンクを案内できる
    • ファイルの変更履歴も見やすくなる

    2. 新しいリポジトリ krm を作る手順（GitHub で操作） #

    1. GitHub にログイン
    2. 右上の「＋」メニューから [New repository] をクリック
    3. リポジトリ名に krm を入力
    4. 説明文（Description）は任意で入力
    5. 「Public」を選ぶ（誰でも見られるようにする）
    6. README.md にチェックを入れて初期化
    7. 「Create repository」で作成完了

    3. ローカルで作業 → コミット → タグ → GitHub にプッシュ #

    以下の手順で、手元のファイルを shikeda/krm に移して、バージョン管理。

    3-1. 初期クローン # 

    git clone https://github.com/foo/bbb.git
cd bbb

    3-2. ファイルをコピー # 

    cp /path/to/old_repo/bbb*.tsv .

    3-3. Git に反映 # 

    git add .
git commit -m "Initial commit of bbb version 1.2"

    3-4. タグを作る（v1.2） # 

    git tag v1.2

    3-5. GitHub に反映 # 

    git push origin main
git push origin v1.2

    4. GitHub 上でリリースページを作成 #

    1. https://github.com/foo/bbb にアクセス
    2. 上部メニューの「Releases」→「Draft a new release」
    3. 「Tag version」に v1.2 を選ぶ
    4. タイトル・説明（例：「初版」や「仕様変更を含む」など）を入力
    5. 「Publish release」で公開

    5. スクリプト化（簡易的な例） #

    たとえば、今後新しいバージョンを出すたびに使えるシェルスクリプトを用意できます：

    #!/bin/bash
VERSION=$1

if [ -z "$VERSION" ]; then
  echo "Usage: ./release.sh v1.3"
  exit 1
fi

git add .
git commit -m "Update files for $VERSION"
git tag $VERSION
git push origin main
git push origin $VERSION

    上のように release.sh を作っておけば、

    ./release.sh v1.3

    で更新＆タグ付けができます。

    補足：foo/AAA からの移行案 #

    • foo/AAA にあった旧ファイル（v1.1 以前）はそのままアーカイブ扱いでもよいです。
    • README に「bbb の最新版は https://github.com/foo/bbb に移動しました」と書いておくと親切です。
    • 不要なら、foo/AAA から bbb.tsv を削除しても構いません。

    なぜ v1.2.9 のように刻むのが良いのか？ #

    • 軽微な修正・改良を反映したタイミングでリリースしておくと、
      • どこで何が変わったか追いやすくなる
      • 複数人で作業していても食い違いが少ない
      • 公開利用者にも「最新版はどれか」がすぐわかる
    • GitHub のリリースページで、バージョンごとの説明文を残しておくことも可能です

    運用のベストプラクティス（参考）

    バージョン番号 用途の目安
    v1.2.0 大きな仕様変更やファイル構成の変更があったとき
    v1.2.1 ～ v1.2.9 小さな修正（文字の訂正、行追加、ラベル変更など）

    ※ 世の中の通例として「メジャー.マイナー.パッチ」というルールに沿っています（Semantic Versioning）。

    release.sh もこのままで OK

    そのまま：

    ./release.sh v1.2.9

    と使ってよいですし、必要があれば次のように履歴メッセージを追加するような拡張も可能です：

    #!/bin/bash
VERSION=$1
MESSAGE=$2

if [ -z "$VERSION" ]; then
  echo "Usage: ./release.sh v1.2.9 \"Fix typo in bbb_yyy.tsv\""
  exit 1
fi

git add .
git commit -m "${MESSAGE:-Update for $VERSION}"
git tag $VERSION
git push origin main
git push origin $VERSION

    こうすると：

    ./release.sh v1.2.9 "Fix typo in line 103 of bbb_xxx.tsv"

    のように記録できます。

    補足：GitHubリリースページも活用を #

    GitHub 上で：

    • 各タグ（例：v1.2.6, v1.2.7…）に対応する「Release」を作っておく
    • 説明欄に「追加した行」「修正箇所」などを書く と、外部の人にもとても親切になります。

    ステップバイステップ：Zenodoで bbb/ccc を登録する手順 #

    1. Zenodoアカウント作成
    • GitHub アカウントでログイン可能（OAuth認証）
    1. GitHubとZenodoを連携
    • Zenodoの GitHub連携設定ページ にアクセス
    • 対象となる GitHub リポジトリ（例：foo/bbb, foo/ccc）を「ON」にする
    1. リリースを作成
    • foo/bbb → v1.2.9 のようにタグをつけて GitHub リリース
    • foo/ccc → v0.9.1 のように別タグでリリース
    1. Zenodoが自動で反応・DOI発行
    • ZenodoがGitHubのリリースを検知し、数分で登録される
    • 各リリースに対応した DOI が発行される
      • 例：v1.2.9 → 10.5281/zenodo.1234567

    補足：ZenodoのDOIの種類

    DOIの種類 用途
    Concept DOI 1つのリポジトリに紐づく「固定のDOI」
    Versioned DOI 各バージョン（例：v1.2.9）ごとの固有のDOI

    リリースノートのテンプレート # 

    ## 📌 バージョン v1.2.3 - 2025-05-20

### ✨ 変更内容（Changes）

- `bbb_yyy.tsv` を v1.2.3 に更新しました。
  - 意義注の追加：行1234「□□□」に補注を追記
  - 音注の修正：行5678「〇〇〇」の読みを修正
  - 書字訂正：誤記「勇」→「勇」を修正

※ `bbb_xxx.tsv` は v1.2.1 のままで変更はありません。

### 📁 更新対象ファイル一覧

| ファイル名       | バージョン | 主な変更       |
|------------------|------------|----------------|
| `bbb_yyy.tsv`     | v1.2.3     | 音注・意義注の修正・追加 |
| `bbb_xxx.tsv`     | v1.2.1     | （変更なし）   |
